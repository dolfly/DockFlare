# Contribuer à DockFlare

Tout d’abord, merci d’avoir envisagé de contribuer à DockFlare ! Les contributions de la communauté sont incroyablement précieuses. Ce document fournit quelques lignes directrices pour contribuer.

## Comment vous pouvez aider

* **Signaler des bugs :** Ouvrez un **Issue** sur GitHub avec les étapes détaillées pour reproduire le bug.
* **Suggérer des améliorations et des idées :** Démarrez une **Discussion** dans l'onglet "Discussions" du référentiel GitHub.
* **Amélioration de la documentation :** Vos commentaires ou demandes d'extraction pour améliorer la documentation sont les bienvenus.
* **Soumission de demandes de tirage :** Si vous souhaitez contribuer au code, veuillez suivre les directives ci-dessous.

## Directives pour les demandes de tirage

1. **Discutez d'abord :** Pour des changements importants, veuillez d'abord ouvrir un problème ou démarrer une discussion pour vous aligner sur l'approche.
2. **Branche :** Créez une nouvelle branche pour votre fonctionnalité ou votre correctif de bogue (`feature/your-feature` ou `fix/your-bug`).
3. **Style de code :** Suivez PEP 8 pour Python. Utilisez des noms de variables et de fonctions clairs.
4. **Test :** Testez minutieusement vos modifications et décrivez le processus de test dans votre pull request.
5. **Documentation :** Mettez à jour le `README.md` ou cette documentation si vos modifications affectent les fonctionnalités destinées aux utilisateurs.
6. **Soumettez le PR :** Ouvrez une demande d'extraction sur la branche `unstable` du référentiel DockFlare principal. Fournissez une description claire de vos modifications.

## Communauté et assistance

* Pour les **rapports de bogues**, veuillez utiliser le tracker [GitHub Issues](https://github.com/ChrispyBacon-dev/DockFlare/issues).
* Pour les **demandes de fonctionnalités, questions, idées ou discussions générales**, veuillez utiliser le forum [GitHub Discussions](https://github.com/ChrispyBacon-dev/DockFlare/discussions).
