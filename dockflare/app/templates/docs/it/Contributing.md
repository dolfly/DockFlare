# Contribuire a DockFlare

Prima di tutto, grazie per aver considerato di contribuire a DockFlare! I contributi della comunità sono incredibilmente preziosi. Questo documento fornisce alcune linee guida per contribuire.

## Come puoi aiutare

* **Segnalazione di bug:** Apri un **problema** su GitHub con i passaggi dettagliati per riprodurre il bug.
* **Suggerimento di miglioramenti e idee:** Avvia una **Discussione** nella scheda "Discussioni" del repository GitHub.
* **Miglioramento della documentazione:** Il tuo feedback o richieste pull per migliorare la documentazione sono benvenuti.
* **Invio di richieste pull:** Se desideri contribuire con il codice, segui le linee guida riportate di seguito.

## Linee guida per le richieste pull

1. **Discuti prima:** per modifiche significative, apri prima un problema o avvia una discussione per allinearti all'approccio.
2. **Diramazione:** crea un nuovo ramo per la tua funzionalità o correzione di bug (`feature/your-feature` o `fix/your-bug`).
3. **Stile codice:** Segui PEP 8 per Python. Utilizza nomi chiari di variabili e funzioni.
4. **Test:** testa attentamente le modifiche e descrivi il processo di test nella richiesta pull.
5. **Documentazione:** aggiorna `README.md` o questa documentazione se le modifiche influiscono sulla funzionalità rivolta agli utenti.
6. **Invia il PR:** Apri una richiesta pull contro il ramo `unstable` del repository DockFlare principale. Fornisci una descrizione chiara delle modifiche.

## Comunità e supporto

* Per le **segnalazioni di bug**, utilizza il tracker [Problemi GitHub](https://github.com/ChrispyBacon-dev/DockFlare/issues).
* Per **richieste di funzionalità, domande, idee o discussioni generali**, utilizza la bacheca [Discussioni su GitHub](https://github.com/ChrispyBacon-dev/DockFlare/discussions).
