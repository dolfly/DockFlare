# Contribuyendo a DockFlare

En primer lugar, ¡gracias por considerar contribuir a DockFlare! Las contribuciones de la comunidad son increíblemente valiosas. Este documento proporciona algunas pautas para contribuir.

## Cómo puedes ayudar

* **Reportar errores:** Abra un **Problema** en GitHub con pasos detallados para reproducir el error.
* **Sugerencias de mejoras e ideas:** Inicie una **Discusión** en la pestaña "Discusiones" del repositorio de GitHub.
* **Mejora de la documentación:** Sus comentarios o solicitudes de extracción para mejorar la documentación son bienvenidos.
* **Envío de solicitudes de extracción:** Si desea contribuir con código, siga las pautas a continuación.

## Directrices para solicitudes de extracción

1. **Discutir primero:** Para cambios significativos, abra un problema o inicie una discusión primero para alinearse con el enfoque.
2. **Bifurcación:** Cree una nueva rama para su función o corrección de errores (`feature/your-feature` o `fix/your-bug`).
3. **Estilo del código:** Siga PEP 8 para Python. Utilice nombres claros de variables y funciones.
4. **Pruebas:** Pruebe sus cambios minuciosamente y describa el proceso de prueba en su solicitud de extracción.
5. **Documentación:** Actualice `README.md` o esta documentación si sus cambios afectan la funcionalidad de cara al usuario.
6. **Envíe el PR:** Abra una solicitud de extracción en la rama `unstable` del repositorio principal de DockFlare. Proporcione una descripción clara de sus cambios.

## Comunidad y soporte

* Para **informes de errores**, utilice el rastreador [Problemas de GitHub](https://github.com/ChrispyBacon-dev/DockFlare/issues).
* Para **solicitudes de funciones, preguntas, ideas o debates generales**, utilice el tablero [Discusiones de GitHub](https://github.com/ChrispyBacon-dev/DockFlare/discussions).

Este es un resumen del `CONTRIBUTING.md` completo. Para obtener más detalles, consulte el [archivo original](https://github.com/ChrispyBacon-dev/DockFlare/blob/main/CONTRIBUTING.md) en el repositorio.
