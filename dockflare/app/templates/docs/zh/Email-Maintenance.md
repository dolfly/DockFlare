# 维护与故障排除

DockFlare Email 的设计目标是低维护成本，但了解备份处理方式和常见问题对于长期可靠性至关重要。

## 备份与恢复

所有邮件数据均存储在 Docker 卷 `mail_data` 中。执行备份的方法：

1.  **完整卷备份：** 备份主机上整个卷文件夹。这是最安全的方式，可完整捕获原始 SQLite 数据库和所有附件文件。
2.  **界面备份：** 在 **邮件** 页面找到 **备份与恢复** 卡片，点击 **下载备份**。系统将生成邮件数据的 ZIP 压缩包。注意：此备份包含明文格式的邮件和附件，请妥善保管。

恢复方法：
1.  确保 `mail_data` 卷已挂载到 `docker-compose.yml` 中。
2.  在 **邮件** 页面的 **备份与恢复** 卡片中，选择 ZIP 文件并点击 **恢复备份**。此操作将永久覆盖现有邮件数据。

## 日志

调试投递问题通常需要查看 `dockflare-mail-manager` 容器的日志。

```bash
docker logs -f dockflare-mail-manager
```

邮件页面还包含 **投递日志** 卡片。点击 **查看** 打开日志查看器，其中有两个标签：
*   **出站日志：** 所有出站邮件尝试的历史记录。
*   **退件日志：** 您发送的邮件中所有投递失败（NDR）的历史记录。

## 高可用性与自愈

### R2 缓冲
如果您的服务器离线（如停电、网络中断），Cloudflare Inbound Worker 将检测到本地 Webhook 不可达，并将邮件安全保存在 **R2 临时缓存**中。
*   Worker 每 5 分钟执行一次 **Cron 任务**。
*   服务器恢复后，自动重试投递所有缓冲邮件。

### 文件系统一致性
Mail Manager 包含一个启动例程，确保数据库与文件系统保持同步。如果附件文件存在但没有对应的数据库记录（「孤立文件」），将自动清除以节省空间。

## 常见问题

### 日志中出现「Worker Error」
确保您的 API 令牌具有 `Workers Scripts` 和 `Workers KV Storage` 权限。如果您最近更新了 DockFlare，可能需要在邮件页面点击 **重新部署 Worker** 以同步新的环境变量。

### 邮件投递延迟
请检查 Cloudflare Worker 控制台中的 **Cron** 日志。如果本地服务器负载较高或存在网络问题，Worker 将把邮件缓冲到 R2，待服务器响应后再进行投递。
