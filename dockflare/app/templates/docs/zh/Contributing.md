# 为 DockFlare 做出贡献

首先，感谢您考虑为 DockFlare 做出贡献！社区贡献非常有价值。本文档提供了一些贡献指南。

## 您可以如何提供帮助

* **报告错误：** 在 GitHub 上打开一个 **问题**，其中包含重现错误的详细步骤。
* **建议增强功能和想法：** 在 GitHub 存储库的“讨论”选项卡上开始 **讨论**。
* **改进文档：** 欢迎您提供反馈或请求以改进文档。
* **提交 Pull 请求：** 如果您想贡献代码，请遵循以下指南。

## Pull 请求指南

1. **首先讨论：** 对于重大更改，请先打开问题或开始讨论以协调方法。
2. **分支：** 为您的功能或错误修复创建一个新分支（`feature/your-feature` 或 `fix/your-bug`）。
3. **代码风格：** 遵循 Python 的 PEP 8。使用清晰的变量和函数名称。
4. **测试：** 彻底测试您的更改并在拉取请求中描述测试过程。
5. **文档：** 如果您的更改影响面向用户的功能，请更新 `README.md` 或本文档。
6. **提交 PR：** 针对主 DockFlare 存储库的 `unstable` 分支打开拉取请求。清楚地描述您的更改。

## 社区与支持

* 对于 **错误报告**，请使用 [GitHub Issues](https://github.com/ChrispyBacon-dev/DockFlare/issues) 跟踪器。
* 对于**功能请求、问题、想法或一般讨论**，请使用 [GitHub 讨论](https://github.com/ChrispyBacon-dev/DockFlare/discussions) 板。

这是完整 `CONTRIBUTING.md` 的摘要。有关更多详细信息，请参阅存储库中的[原始文件](https://github.com/ChrispyBacon-dev/DockFlare/blob/main/CONTRIBUTING.md)。
