# Wkład w DockFlare

Po pierwsze, dziękuję za rozważenie wniesienia wkładu w DockFlare! Wkład społeczności jest niezwykle cenny. Ten dokument zawiera pewne wytyczne dotyczące wnoszenia wkładu.

## Jak możesz pomóc

* **Zgłaszanie błędów:** Otwórz **Problem** w GitHub ze szczegółowymi instrukcjami, jak odtworzyć błąd.
* **Sugerowanie ulepszeń i pomysłów:** Rozpocznij **dyskusję** na karcie „Dyskusje” w repozytorium GitHub.
* **Poprawa dokumentacji:** Mile widziane są Twoje uwagi lub prośby o ulepszenie dokumentacji.
* **Przesyłanie żądań ściągnięcia:** Jeśli chcesz przesłać kod, postępuj zgodnie z poniższymi wskazówkami.

## Wytyczne dotyczące żądań ściągnięcia

1. **Najpierw przedyskutuj:** W przypadku znaczących zmian otwórz najpierw problem lub rozpocznij dyskusję, aby uzgodnić podejście.
2. **Rozgałęzianie:** Utwórz nową gałąź dla swojej funkcji lub poprawki błędu (`feature/your-feature` lub `fix/your-bug`).
3. **Styl kodu:** Postępuj zgodnie z PEP 8 dla Pythona. Używaj przejrzystych nazw zmiennych i funkcji.
4. **Testowanie:** Dokładnie przetestuj wprowadzone zmiany i opisz proces testowania w żądaniu ściągnięcia.
5. **Dokumentacja:** Zaktualizuj `README.md` lub tę dokumentację, jeśli zmiany wpływają na funkcjonalność dostępną dla użytkownika.
6. **Prześlij żądanie ściągnięcia:** Otwórz żądanie ściągnięcia dla gałęzi `unstable` głównego repozytorium DockFlare. Podaj jasny opis zmian.

## Społeczność i wsparcie

* Aby uzyskać **raporty o błędach**, użyj narzędzia do śledzenia [Problemy z GitHub](https://github.com/ChrispyBacon-dev/DockFlare/issues).
* W przypadku **próśb o nowe funkcje, pytań, pomysłów lub ogólnych dyskusji** należy skorzystać z tablicy [Dyskusje w GitHub](https://github.com/ChrispyBacon-dev/DockFlare/discussions).

To jest podsumowanie pełnego kodu `CONTRIBUTING.md`. Więcej szczegółów znajdziesz w [oryginalnym pliku](https://github.com/ChrispyBacon-dev/DockFlare/blob/main/CONTRIBUTING.md) w repozytorium.
