# Berkontribusi ke DockFlare

Terima kasih sudah mempertimbangkan untuk berkontribusi ke DockFlare. Kontribusi komunitas sangat berharga. Dokumen ini memberikan beberapa panduan dasar.

## Cara Anda Bisa Membantu

*   **Melaporkan bug:** buka **Issue** di GitHub dengan langkah reproduksi yang jelas
*   **Mengusulkan enhancement & ide:** mulai **Discussion** di tab Discussions pada repository GitHub
*   **Memperbaiki dokumentasi:** feedback dan pull request dokumentasi selalu diterima
*   **Mengirim pull request:** jika ingin berkontribusi kode, ikuti pedoman di bawah

## Pedoman Pull Request

1. **Diskusikan dulu:** untuk perubahan besar, buka Issue atau Discussion terlebih dahulu
2. **Branching:** buat branch baru untuk fitur atau bug fix
3. **Code style:** ikuti PEP 8 untuk Python
4. **Testing:** uji perubahan Anda dengan baik dan jelaskan proses testing di pull request
5. **Dokumentasi:** perbarui `README.md` atau dokumentasi ini jika perubahan memengaruhi fitur yang terlihat oleh pengguna
6. **Kirim PR:** buka Pull Request ke branch `unstable` pada repository utama DockFlare

## Komunitas & Dukungan

*   Untuk **bug report**, gunakan [GitHub Issues](https://github.com/ChrispyBacon-dev/DockFlare/issues)
*   Untuk **feature request, pertanyaan, ide, atau diskusi umum**, gunakan [GitHub Discussions](https://github.com/ChrispyBacon-dev/DockFlare/discussions)

Ini adalah ringkasan dari `CONTRIBUTING.md` penuh. Untuk detail lengkap, lihat [file aslinya](https://github.com/ChrispyBacon-dev/DockFlare/blob/main/CONTRIBUTING.md).
