# Mitwirken an DockFlare

Zunächst einmal: Vielen Dank, dass Sie in Erwägung ziehen, zu DockFlare beizutragen! Beiträge aus der Community sind unglaublich wertvoll. In diesem Dokument finden Sie einige Richtlinien für Ihre Mitarbeit.

## Wie Sie helfen können

*   **Fehler melden:** Öffnen Sie ein **Issue** auf GitHub mit detaillierten Schritten zur Reproduktion des Fehlers.
*   **Verbesserungen & Ideen vorschlagen:** Starten Sie eine **Discussion** im Tab "Discussions" des GitHub-Repositories.
*   **Dokumentation verbessern:** Ihr Feedback oder Pull Requests zur Verbesserung der Dokumentation sind herzlich willkommen.
*   **Pull Requests einreichen:** Wenn Sie Code beisteuern möchten, befolgen Sie bitte die folgenden Richtlinien.

## Richtlinien für Pull Requests

1.  **Zuerst diskutieren:** Bei größeren Änderungen öffnen Sie bitte zuerst ein Issue oder starten Sie eine Discussion, um den Lösungsansatz abzustimmen.
2.  **Branching:** Erstellen Sie einen neuen Branch für Ihr Feature oder Ihren Bugfix (`feature/ihr-feature` oder `fix/ihr-bug`).
3.  **Code-Stil:** Folgen Sie PEP 8 für Python. Verwenden Sie klare Variablen- und Funktionsnamen, verzichten Sie strikt auf inline Kommentare in den App-Modulen selbst.
4.  **Testen:** Testen Sie Ihre Änderungen gründlich und beschreiben Sie den Testprozess in Ihrem Pull Request.
5.  **Dokumentation:** Aktualisieren Sie die `README.md` oder diese Dokumentation, falls Ihre Änderungen die benutzerseitige Funktionalität betreffen.
6.  **PR einreichen:** Öffnen Sie einen Pull Request gegen den `unstable`-Branch des DockFlare-Hauptrepositories. Geben Sie eine klare Beschreibung Ihrer Änderungen an.

## Community & Support

*   Für **Fehlerberichte** verwenden Sie bitte den Tracker unter [GitHub Issues](https://github.com/ChrispyBacon-dev/DockFlare/issues).
*   Für **Feature-Anfragen, Fragen, Ideen oder allgemeine Diskussionen** nutzen Sie bitte das [GitHub Discussions](https://github.com/ChrispyBacon-dev/DockFlare/discussions)-Board.

Dies ist eine Zusammenfassung der vollständigen `CONTRIBUTING.md`. Weitere Details finden Sie in der [Originaldatei](https://github.com/ChrispyBacon-dev/DockFlare/blob/main/CONTRIBUTING.md) im Repository.
