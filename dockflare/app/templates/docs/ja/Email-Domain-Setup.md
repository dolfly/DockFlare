# ドメインのセットアップと設定

`email` プロファイルで Docker コンテナが起動したら、DockFlare Web UI で自動セットアッププロセスを開始できます。

## メール設定ウィザード

1.  左サイドバーの **メール** ページに移動します。
2.  **メールドメインをセットアップ** をクリックします。
3.  設定したい **Cloudflare ゾーン** (ドメイン) を選択します。
4.  **セットアップを確認** をクリックします。

### セットアップ中に何が起こるか?
DockFlare は Cloudflare API を通じていくつかの自動化されたステップを実行します。
*   ゾーンの **Email Routing を有効化** します。
*   **DNS を設定します:** Cloudflare Email Routing が必要とする MX レコード、SPF (TXT)、DMARC (TXT)、DKIM (CNAME) レコードを作成します。
*   **ストレージをプロビジョニングします:** 一時的なトランジットバッファリング用の専用 R2 バケットを作成します。
*   **ワーカーをデプロイします:** メール受信用の Inbound Worker とメール送信用の Outbound Worker をデプロイします。
*   **KV を初期化します:** エッジでメールボックスのクォータを追跡するための Cloudflare KV 名前空間を作成します。

## DNS の正常性を確認する

DNS の変更が伝播するには時間がかかることがあります。メールページには **DNS レコード** カードが表示されます。
*   **DNS を確認** をクリックして、MX、SPF、DMARC レコードの現在のステータスを確認します。(DKIM は Cloudflare Email Routing によって自動的に管理されており、ここでは個別に確認されません。)
*   レコードがパブリック DNS で正しく検出されると、システムに緑色のバッジが表示されます。

## ワーカーの更新 / 再デプロイ

DockFlare のバージョンを更新したり、API 権限を変更したりした場合、ワーカーを更新する必要がある場合があります。
*   **ワーカーを再デプロイ** ボタンをクリックします。
*   これにより、最新のワーカーロジックが再アップロードされ、保存されたメールデータに影響を与えることなく、すべてのバインディング (R2、KV、Webhook シークレット) が再同期されます。

## ドメインの削除

ドメインのメールホスティングを停止したい場合:
*   **ドメインを削除** をクリックします。
*   これにより、ルーティングルール、Inbound/Outbound ワーカー、R2 トランジットバケット、および DNS レコードが Cloudflare から削除されます。
*   **注意:** これにより `mail_data` ボリューム内のローカルメールデータが*削除されるわけではありません*。サーバーに保存されているメッセージと添付ファイルも削除したい場合は、削除ダイアログで **ローカルデータを含める** を有効にしてください。
