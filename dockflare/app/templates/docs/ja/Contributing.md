# DockFlare に貢献する

DockFlare への貢献をご検討いただきありがとうございます。コミュニティの協力はプロジェクトにとって非常に重要です。このページでは、貢献方法のガイドラインをまとめます。

## あなたができること

* **バグの報告:** GitHub で **問題** を開き、バグを再現する詳細な手順を記載します。
* **機能強化とアイデアの提案:** GitHub リポジトリの Discussions でディスカッションを開始します。
* **ドキュメントの改善:** ドキュメントを改善するためのフィードバックやプルリクエストを歓迎します。
* **プル リクエストの送信:** コードを提供したい場合は、以下のガイドラインに従ってください。

## プル リクエストのガイドライン

1. **最初に話し合ってください:** 重要な変更については、最初に問題を開くか、ディスカッションを開始してアプローチを調整してください。
2. **分岐:** 機能またはバグ修正用に新しい分岐を作成します (`feature/your-feature` または `fix/your-bug`)。
3. **コード スタイル:** Python の PEP 8 に従います。明確な変数名と関数名を使用します。
4. **テスト:** 変更を徹底的にテストし、プル リクエストにテスト プロセスを記述します。
5. **ドキュメント:** 変更がユーザー向けの機能に影響を与える場合は、`README.md` またはこのドキュメントを更新します。
6. **PR を送信します:** メインの DockFlare リポジトリの `unstable` ブランチに対してプル リクエストを開きます。変更内容を明確に説明してください。

## コミュニティとサポート

* **バグ レポート**については、[GitHub Issues](https://github.com/ChrispyBacon-dev/DockFlare/issues) トラッカーを使用してください。
* **機能のリクエスト、質問、アイデア、または一般的なディスカッション**については、[GitHub Discussions](https://github.com/ChrispyBacon-dev/DockFlare/discussions) ボードをご利用ください。

これは `CONTRIBUTING.md` の要約です。詳しくはリポジトリ内の [CONTRIBUTING.md](https://github.com/ChrispyBacon-dev/DockFlare/blob/main/CONTRIBUTING.md) を参照してください。
